# 萱萱单词闯关 v6.00 — 错题本产品方案

**版本**: v6.00 | **状态**: 对齐确认中 | **日期**: 2026-05-19

---

## 一、术语统一

| 术语 | 定义 |
|------|------|
| **错题本** | 唯一入口，唯一名称。进入后直接开始专项答题（消灭模式）。 |
| **pending** | 待消灭错题（后台术语，不对用户展示）。 |
| **eliminated** | 已消灭错题（后台术语，不对用户展示）。 |

---

## 二、核心原则

1. **零操作收录** — 答错自动入库，不弹窗、不中断、不问用户。
2. **直达答题** — 点击"错题本"入口后，没有列表/大本营，直接抽题开答。
3. **小批次歼灭** — 每轮最多 10 题，短平快，不疲劳。
4. **即时正反馈** — 答对即消灭，结果页放烟花。
5. **隐性保险** — 系统后台记录 wrongCount，主线/大乱斗再错自动回归 pending。

---

## 三、数据模型

```javascript
// 纯前端 localStorage，与现有 xx_progress 共存
wrongBook = {
  pending: [
    {
      word: "apple",
      correct: "苹果",
      pos: "n.",
      example: "I eat an **apple** every day.",
      wrongCount: 2,        // 累计答错次数
      lastWrong: "香蕉",   // 最近一次错选
      lastTime: "2026-05-19T10:23:00"  // 最近一次答错时间
    }
  ],
  eliminated: [
    {
      word: "banana",
      correct: "香蕉",
      pos: "n.",
      eliminatedAt: "2026-05-19T10:30:00",
      wrongCount: 3        // 历史累计（用于分析）
    }
  ]
}
```

## 四、收录规则

### 自动收录（答错即入库）

| 场景 | 是否收录 | 说明 |
|------|---------|------|
| 主线闯关答错 | ✅ 收录 | 核心学习场景，错即暴露盲区。 |
| 大乱斗答错 | ✅ 收录 | 全词库大乱斗中暴露的盲区同样重要。 |
| 复习模式答错 | ❌ 不收 | 复习就是用来试错的，不收避免污染错题本。 |
| 错题本（消灭模式）答错 | ❌ 不收 | 已在 pending 中，只需更新 wrongCount。 |

### 收录逻辑

1. 若单词已在 pending → `wrongCount++`，更新 `lastWrong` 和 `lastTime`。
2. 若单词已在 eliminated → 移回 pending，`wrongCount++`（**隐性保险触发**）。
3. 若单词不在任何列表 → 新增到 pending，`wrongCount=1`。

## 六、错题本流程（消灭模式）

### Step 1：点击入口 → 进入错题本启动页

**有错题时：**
- 标题："错题歼灭战"
- 副标题："从 7 个待消灭单词中随机抽取"
- 主按钮："🔥 开始歼灭"（大按钮）
- 次按钮："返回首页"

**无错题时：**
- 标题："太棒了！✨"
- 文案："当前没有错题，继续保持！"
- 按钮："返回首页"

### Step 2：答题

- 从 pending 中随机抽取最多 10 个单词。
- 每个单词生成 6 个选项（1 正确 + 5 干扰）。
- 干扰项从全词库（BANK）随机抽取，不局限于错题。
- UI 完全复用主线闯关答题界面（题目区 + 选项按钮 + 提示按钮）。
- **没有解锁/轮次/正确率门槛概念** — 纯粹练兵。

### Step 3：结果页

**标题区：**
- 根据消灭率动态文案：
  - 100%："全部消灭！🎉" + 烟花动效
  - ≥50%："消灭 X 个敌人！💪"
  - <50%："继续加油！还有 X 个敌人"

**统计区：**
- "本次消灭 7/10"
- "耗时 XX 秒"

**列表区（可折叠）：**
- 已消灭（绿色勾）
- 仍然存活（红色叉 + 正确释义提示）

**操作区：**
- 主按钮："🔥 继续歼灭"（重新抽 10 题，若 pending 不足 10 则有多少抽多少）
- 次按钮："🏠 返回首页"

### Step 4：数据更新

- 答对的单词 → 从 pending 移到 eliminated（消灭成功）。
- 答错的单词 → 留在 pending，`wrongCount++`。

## 七、隐性保险机制

**用户无感知，后台自动运行：**

```
主线闯关 / 大乱斗答错时：
  IF 单词在 eliminated:
    → 移回 pending
    → wrongCount++
    → lastWrong = 本次错选
    → lastTime = 现在
  ELSE IF 单词不在任何列表:
    → 新增到 pending
    → wrongCount=1
```

**意义：**
- 萱萱以为"消灭了"，但如果真没记住，下次主线/大乱斗答错时它会悄悄回来。
- 家长/老师可以通过 `eliminated` 列表的历史 wrongCount 看到"这个单词以前错 3 次才消灭"——暴露顽固盲区。

## 八、与现有系统的关系

| 现有系统 | 关系 |
|---------|------|
| `xx_progress` localStorage | 独立存储 `wrongBook`，不破坏现有进度。 |
| 主线闯关 | 答错即收录；eliminated 单词答错自动复活。 |
| 大乱斗 | 同上。 |
| 复习模式 | 答错不收，避免重复收录。 |
| 排行榜 | 错题本成绩不进排行榜（非竞技场景）。 |
