# 更新日志 v5.15 — 错题本功能

## 新增功能

### 📚 错题本（错题歼灭战）
- **首页入口**：复用现有"📝 错题本"卡片，点击直接进入
  - 有待消灭单词时：显示红色数字徽章 + "待消灭 X 个"
  - 无错题时：显示 "暂无错题"
- **启动页**：显示当前错题统计（待消灭 / 已消灭 / 累计）
  - 有错题 → "🔥 开始歼灭" 按钮
  - 无错题 → "✨ 太棒了！当前没有错题"
- **答题流程**：
  - 从 pending 错题中随机抽取，最多 10 题
  - 复用现有答题界面，顶部进度条正常显示
  - 每题 10 秒倒计时，支持发音和提示
- **结果页**：
  - 消灭率统计 + 已消灭/存活数量
  - 存活列表分组显示（已消灭 / 仍然存活）
  - **100% 消灭 → 烟花动画 + "全部消灭！🎉"**
  - <100% → 温和鼓励文案
  - 还有错题时 → "🔥 继续歼灭"；无错题时 → 按钮隐藏

## 自动收录规则

| 场景 | 答错时 | 备注 |
|------|--------|------|
| **主线闯关** | ✅ 自动收录到 pending | — |
| **大乱斗** | ✅ 自动收录到 pending | — |
| **复习模式** | ❌ 不收 | 避免干扰错题本纯度 |
| **错题本歼灭** | ❌ 不收 | 只更新内部 wrongCount |

## 消灭机制（一次消灭 + 隐形保险）

- **一次消灭**：错题本中答对一次 → 从 pending 移出，加入 eliminated
- **隐形保险**：eliminated 单词在主线/大乱斗中再次答错 → 自动回归 pending
- **wrongCount 跟踪**：累计答错次数，数据持久化到 localStorage

## 数据持久化

- **localStorage 键**：`xx_wrongbook`
- **结构**：`{pending: [...], eliminated: [...]}`
- **自动保存**：每次答题结果、每次消灭完成
- **向后兼容**：v5.14 及以前版本没有此键，首次加载时自动初始化空错题本

## 文案调整

- 全部使用温和措辞，无严厉或催促感
- 存活列表中不显示"上次错选"提示
- 结果页分级：
  - 100% → "全部消灭！🎉" + 烟花
  - ≥50% → "消灭 X 个敌人！💪"
  - <50% → "继续加油！还有 X 个敌人"

## 其他

- 复习模式独立为 `startReviewRound()`，不再与主线闯关共用启动函数
- 新增 `G.isReview` 标志，用于区分主线闯关和复习模式
- 烟花动画 CSS 使用 `@keyframes boom`
